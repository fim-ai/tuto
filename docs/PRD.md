# Tuto 功能书

v3，2026-07-15。定位：中立引文审计机构。KPI 是传播与品牌，收入预期为零。

## 本期范围：V1 + V2 一起做

- **V1 报告**：ACL 2026 主会 + Findings L1 审计 → tuto.fim.ai 上的网页版公开报告 + 匿名化数据集（CC BY）。
- **V2 自助工具**：单篇上传（PDF 或 bib 粘贴）→ 逐条核验报告（措辞用 suspect，不下 fake 定论）→ 问题条目旁给 Cito 修复建议。
- **V3 批量/API**：本期只挂 "Batch audit for institutions — contact us"。
- **不做**：L2 entailment（只存 contexts.jsonl 原料）、MCP、任何点名。

## 站点结构（tuto.fim.ai，Next.js）

1. **首页**：一句话定位 + 最新报告核心数字 + 工具入口
2. **Reports**：一期一页（聚合统计、跨会场纵向指数、L3 demo 互动区、数据集下载）
3. **Check**（V2 工具）：单篇自助核验，调 api.tuto.fim.ai/check
4. **Methodology**：方法论全文、已知局限、抽检误报率±区间、开源仓库链接
5. API 出口：api.tuto.fim.ai（/check、/repair）

报告与工具同域不同路由（`/reports/acl-2026`、`/check`）：报告是流量入口、工具是转化出口，分域会剪断漏斗。

## 审计流程与验收线

1. **采集**：Anthology 全量 bib + PDF 限速下载，断点续传。
2. **解析**：GROBID（`processReferences`）。**不用 MinerU**——ACL 的 PDF 是 born-digital，无 OCR 需求。
3. **解析验收**（改自 v1 的「人工抽 50 篇」）：
   - 全量自动交叉比对：用独立的版式计数器复核 GROBID 是否漏抽，覆盖 4,459 篇，零成本。
   - 定向人工抽检：核对自动比对筛出的 30 篇（15 随机 control + 15 适度分歧 probe），约 1 小时，据此算 GROBID 召回率 ± 区间并公开。
   - 全量实测：GROBID 抽出 209,760 条参考文献，中位数比值 1.0，70.7% 在 ±2 内；probe 已定位真实漏抽（含 6 篇 GROBID 零抽需兜底重跑）。
   - 不过线不进下一步。
4. **L1 核验**：本地快照优先 + API 兜底；四分类；每条 flag 带证据链。
5. **自动化复核漏斗**（人工压缩到一个晚上，详见 ARCHITECTURE）：
   自动抢救 → LLM 仲裁（Haiku，~$5）→ 人工仅三件事：抽检 150 条算误报率、挑 10-20 个 demo 案例、处理作者申诉。
6. **修复 demo**：确认案例 → Cito 检索 → 真实替代 + 现成 bib。报告站互动区呈现，传播核心。

## 发布与通知流程（7 天申诉窗）

1. 漏斗跑完 → 当天向涉事论文通讯作者发邮件：证据链 + Cito 修复建议（邮件即产品投放）+ 申诉通道。
2. 7 天申诉窗；确认误报的从统计移除，误报率公开写进方法论页。
3. 期满发布报告（聚合统计 + 匿名案例，永不点名）。
4. +30 天跟进报告：作者修正率（arXiv 更新数）+ 完整数据集。第二波传播。

## 口径红线

- 「裸 LLM 输出 78-90% 捏造」与「真实论文约 1/458 发生率」永远分开呈现。
- 公开数字 = 漏斗末端 suspect 数 × 抽检 precision 校正，附置信区间；minor-mismatch 单列不计入「幻觉」。
- 引文抽取的召回率必须与幻觉率一起公开：漏抽的引文不会变成 suspect，召回率是所有下游数字的天花板。

## 开源策略（open-core）

- 开：`tuto` 管线（Apache-2.0）、数据集（CC BY）、方法论。公信力来自可复现。
- 闭：`tuto-app`（`api/` 含 repair，依赖 Cito 索引；`web/`）。
